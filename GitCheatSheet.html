Git Commands Cheat Sheet


This document is a practical reference for using Git with with multiple projects on GitHub It assumes you already have git installed and a GitHub account set up with SSH or HTTPS access.

1) Starting from GitHub Cloning Locally

If you begin a project by creating a repository on GitHub first, the next step is to bring that empty repository down to your computer. This ensures that your local folder is already connected to the GitHub remote, so you can immediately start adding files and pushing commits.

bash

# Go to where you want the project folder
cd ~/projects

# Clone empty repo from GitHub
git clone git@github.com:USERNAME/REPO_NAME.git

cd REPO_NAME

# Add your files here, then commit + push
git add . 
git commit -m "Initial commit"
git push -u origin main

2) Start Locally Pushing to GitHub

Sometimes you already have a project folder with files before setting up Git. In this case, you can turn that folder into a Git repository, commit its contents, and then link it to an empty GitHub repo. After linking, your local and remote repositories will stay in sync.

bash

cd ~/projects/local_project

# Initialize git
git init

# Add files
git add .
git commit -m 'Initial commit'

# Create empty repo on GitHub, then link it
git remote add origin git@github.com:USERNAME/REPO_NAME.git

# Push to GitHub
git push -u origin main

3) Daily Workflow (Inside Any Repo Folder)

Once your project is set up, your day-to-day work involves tracking changes, committing them, and syncing with GitHub. This workflow is the same for every repository you manage.

bash
git status                  # check changes
git add .                   # stage all changes
git commit -m 'Message'     # commit snapshot
git push                    # upload to GitHub
git pull                    # update with GitHub changes

4) Switching Between Projects

Each project you work on is independent and has its own .git/ directory. That means Git knows which repository it's working with based on the folder you're currently in. To swtich between projects, simply navigate into the correct direcotry, and Git will apply commands to that project only.

bash

cd ~/projects/projec1       # now working on Project 1
git push              

cd ~/projects/project2      # now switch to working on Porject 2
git push

5) Common Handy Commands

Beyond the basics, Git has a set of frequently usefull commands that help you inspect history, manage branches, and undo changes. These are not part of the minimal daily workflow, but they are extramely handy when you want to look deeper into your repository or fix mistakes.

git log --oneline --graph

- to see changes not staged
git diff

- to discard local changes to a file
git checkout FILE

- to list branches
git branch

  • to create and switch to new branch
    git checkout -b new-branch

Tips to keep in mind

  • Run git status often it — it shows exactly what Git is tracking
  • Write meaningful commit messages (e.g., "Fix bug in plotting function" rather then "Update").
  • one project - one repository. This avoids mixing unrelated work.
